7. RAG — Интерпретация и ответы

Текущее состояние: Vector search → Graph traversal → LLM context assembly → Ollama inference.
┌─────┬─────────────────────────┬───────────────────────────────────────────────────────────────────────────────────────────────────────────┐
│  #  │       Точка роста       │                                                   Суть                                                    │
├─────┼─────────────────────────┼───────────────────────────────────────────────────────────────────────────────────────────────────────────┤
│ 7.1 │ Нет multi-hop reasoning │ Граф обходится на 1 уровень соседей. Для вопросов "как данные проходят от A до Z" нужен глубокий обход    │
├─────┼─────────────────────────┼───────────────────────────────────────────────────────────────────────────────────────────────────────────┤
│ 7.2 │ Нет кеширования ответов │ Одинаковые вопросы каждый раз вызывают полный RAG-пайплайн. Redis используется, но не для RAG-результатов │
├─────┼─────────────────────────┼───────────────────────────────────────────────────────────────────────────────────────────────────────────┤
│ 7.3 │ Таймауты LLM            │ 90с на inference, 120с общий — при больших контекстах Ollama может не успевать, особенно qwen3-coder:30b  │
├─────┼─────────────────────────┼───────────────────────────────────────────────────────────────────────────────────────────────────────────┤
│ 7.4 │ Нет re-ranking          │ Результаты vector search идут напрямую в LLM без переранжирования по релевантности                        │
├─────┼─────────────────────────┼───────────────────────────────────────────────────────────────────────────────────────────────────────────┤
│ 7.5 │ Нет graph-aware RAG     │ Контекст для LLM собирается плоско (чанки), а не с учётом структуры графа (вызовы, зависимости)           │
└─────┴─────────────────────────┴───────────────────────────────────────────────────────────────────────────────────────────────────────────┘
